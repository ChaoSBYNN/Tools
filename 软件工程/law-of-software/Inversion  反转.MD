# Inversion  反转

> 通过考虑相反的结果并从那里倒推来解决问题。

* 对于任何目标，都可以反过来思考一下。如果目标是“按时完成项目”，那么我们应该思考“究竟是什么因素会导致我们无法按时完成？”把那些可能造成延误的因素列出来（比如范围扩大、低估了工作难度等），有助于我们实现目标并避免各种问题。
* 可以采用事前分析之类的方法。我们的项目失败了，我们试图找出导致失败的原因。这种批判性思维有助于发现那些被乐观的规划所忽视的风险。
* 在设计和测试过程中，我们会充分考虑系统中可能出现的各种极端情况和使用场景。例如，恶意用户可能会如何破坏这个 API 呢？通过这种方式进行思考，就能找到更好的解决方案（比如，在了解问题产生的原因后，可以添加验证机制、限制请求频率等措施）。

## Overview  概述

其核心思想是：从相反的角度来审视问题，就能更清晰地理解其中的本质。在软件工程中，不应只考虑最佳情况来设计系统，而应预先考虑最坏的情况：如果数据库发生故障怎么办？如果延迟突然增加又该如何处理？

通过思考各种可能出错的情况，我们可以添加诸如断路器、重试机制或故障转移策略之类的功能。这种思维方式与“第一性原理思维”相得益彰，是一种防御性设计手段，通过提前避免各种潜在问题，从而确保系统的稳定性。

## Examples  例子

Netflix 的“混沌猴”计划就是个典型的例子。通常情况下，工程师们会努力确保系统的正常运行。而“混沌猴”计划则恰恰相反：它会随机关闭生产环境中的服务器，以此来测试系统的弹性。这一做法帮助 Netflix 打造出了能够承受服务器故障的、具有容错能力的系统。

测试驱动开发（TDD）通过先编写无法通过的测试来实现这种“倒置”的开发方式。在编写代码使其通过之前，你需要先定义出测试失败的条件。这样一来，你不得不先思考代码可能出错的情况，然后再进行编码实现。

## Origins  起源

“反向思考”的概念可以用投资者查理·芒格的著名话来解释：“我只想知道自己会在哪里死去，这样我就不会去那个地方。”芒格的这一理念源自 19 世纪的数学家卡尔·古斯塔夫·雅可比。雅可比通过反向思考来解决问题，并提出了“永远采取反向思考的方式”。

在哲学和斯多葛派的实践理念中，我们也可以看到一种“反向思考”的方式。斯多葛派有一种名为“预想最坏情况”的练习，也就是事先想象可能发生的最糟糕的事情，从而做好应对准备。这本质上就是将反向思考的理念应用到了生活中。
