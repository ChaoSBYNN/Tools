# Zawinski's Law  扎温斯基定律

> 软件膨胀
> 每个程序都会尝试扩展，直到能够读取邮件。

* 功能膨胀是不可避免的。随着时间推移，软件往往会积累更多功能，导致软件臃肿。
* 一个精简、极简的应用随着流行度不断增加，直到变得和竞争对手一样复杂。
* 程序扩展是因为用户（和产品经理）不断要求“再多一个功能”。人们不断面临整合流行功能以阻止用户转向其他工具的压力。
* 每增加一个新功能，都会增加复杂度，使产品对用户来说更加困惑。开发者应保护该工具的重点，抵制平台蔓延。

## Overview  概述

扎温斯基定律是一个关于软件进化的幽默观察，指出应用程序会不断获得功能，直到它们做了所有事情，甚至是完全超出其原始范围的事情。它强调了功能膨胀，即软件开发领域范围的逐步扩展。

随着应用吸引更多用户，它面临着越来越多的期待，需要增加更多功能。一个基础的笔记应用后来可能会加入聊天或分享功能。

扎温斯基的观点是关于“平台化”：一旦用户在一天中大量时间都生活在一个应用中，就会有压力要求该应用成为一个能做一切的平台。不受控制的扩张可能会破坏产品的原始价值。添加功能很容易，但只添加合适的功能并对其余部分说“不”是必须的。

## Examples  示例

Netscape Navigator 从一个轻薄的浏览器发展成为 Netscape Communicator，一个涵盖浏览器、电子邮件、新闻和网页编辑功能的庞大套件。它变得迟缓且过于复杂，为 Firefox 铺平了道路，Firefox 故意简化为仅仅快速的浏览器。Firefox 本身后来插件和主题变得更丰富。

Slack 的初衷是“终止邮件”，但实际上整合了语音通话、视频会议、文件共享、机器人和应用插件。现在它希望成为一个一站式的工作场所中心，远远超越了简单的信息传递。

GitHub 最初托管代码，后来扩展到问题跟踪、维基、项目板块、讨论、CI 流水线和包注册。

## Origins  起源

Jamie Zawinski（简称 jwz）大约在 1995 年任职 Netscape 期间制定了这条定律。他是 Netscape Navigator 的关键程序员，后来又添加了集成的 Netscape Mail 阅读器。

他将浏览器的演变描述为“我们对软件包围定律证明的贡献”。Netscape 最初是一个网页浏览器，但到了 2.0-3.0 版本，扩展到包含了电子邮件客户端和新闻阅读器。

“阅读邮件”之所以被选为例子，是因为在90年代中期，你经常需要退出当前应用，单独启动邮件程序。
