# The Lindy Effect  林迪效应

> 一件东西使用的时间越长，它继续被使用的可能性就越大。

* 林迪效应指出，如果某事物（一项技术、工具或概念）已经存在很长时间，它很可能还会持续更长时间。
* 对于开发者来说，林迪定律提醒他们投资于永恒的技能和经过验证的基础（算法、核心语言、设计原则），而不是追逐那些可能在几年内就会过时的每一个新框架。
* 在评估技术时使用林迪效应。存在时间较长的事物在未来会持续更长时间（也就是说，选择无聊的技术）。

## Overview  概述

林迪效应颠覆了我们对衰老的直觉。对于生物来说，年长意味着寿命减少。但在技术领域，情况往往相反：使用时间越长，继续使用的可能性就越大。

时间就像一个过滤器。任何不相关、有缺陷或脆弱的东西都倾向于消失，而经过多年后留存下来的通常是高质量且有价值的东西。

这一原则可以作为对抗“闪亮对象综合症”的现实检验。所谓的“创新”大多数与当代思想关系不大；相反，它们只是对过去想法的重新发明。软件中真正基础的思想（算法、数据结构、协议、范式）变化缓慢。

应用林迪定律意味着要偏爱那些“经得起时间考验”的事物，学习像 Python、Java 或 C 这样的语言，以及算法，而不是过度关注最热门的新框架。

## Examples  例子

COBOL 是一种源于 1959 年的语言，已有 60 多年的历史，但它仍然支撑着世界上大量金融和政府基础设施。据一些估计，全球 70-80%的所有金融交易都依赖于银行系统和大型机中的 COBOL 代码。

过去十年中，许多 JavaScript 框架如 AngularJS、Ember.js 和 Knockout.js 兴起又衰落。然而，底层语言和浏览器 API 已经存在了 25 年以上，并将继续存在。

在学习尖端框架和 SQL 之间进行选择时，Lindy 方法建议 SQL（来自 20 世纪 70 年代）至今仍然高度相关，并将继续保持。

## Origins  起源

该术语由记者阿尔伯特·高德曼于 1964 年提出，以纽约市著名的娱乐人士聚集地“林迪熟食店”命名。那里的喜剧演员观察到，百老汇剧目的持续时间可以预测其未来的演出长度：如果一个剧目持续了 X 周，那么可以预期它还会再持续 X 周。

纳西姆·尼古拉斯·塔勒布在他的《黑天鹅》（2007 年）和《反脆弱》（2012 年）两本书中将其介绍给了广大读者，并称之为“林迪效应”，以描述书籍、思想、技术或传统等非易腐物品的陈化。
